# deploy :: shared spec

Единая система для всех сценариев команды `deploy`. Опирается на
`autorun_state_cheatsheet.md` (source of truth для маркеров и
анимаций). Здесь — только deploy-специфичное.

---

## 1. pipeline одного шага

каждый «шаг деплоя» — это пара строк: spinner + result. строка
перерисовывается на месте (не добавляет новую).

```
[/] running tests                 ← spinner, min 600ms
[*] running tests .............. 142/142   ← result
```

возможные концовки spinner-строки:
- `[*]` — шаг успешно завершён
- `[!]` — предупреждение, но движемся дальше
- `[=]` — шаг осознанно пропущен
- `[-]` — шаг на паузе, вернёмся
- `[x]` — abort, pipeline останавливается или меняет ветку

spinner никогда не остаётся на экране финальной строкой — всегда
трансформируется.

---

## 2. структура строки

см. cheatsheet §6. короткое напоминание:

```
[X] <text>................................ <suffix>
```

суффиксы для deploy:
- число / счётчик: `142/142`, `1.2mb`, `23 locations`
- результат: `ok`, `done`, `accepted`, `skipped`, `n/a`
- время: `30s`, `3m 14s`
- пусто — когда строка самодостаточна (`[!] it's friday 17:42`)

---

## 3. анимации: deploy-specific usage

| анимация | применять когда                                        |
|----------|--------------------------------------------------------|
| spinner  | любой `[/]` шаг (по умолчанию)                         |
| dots     | вместе со spinner, по дефолту                          |
| typeout  | прямая речь, цитаты из slack, человеческие реплики     |
| flash    | `[!]` warning и `[x]` abort, всегда                    |
| shake    | `[x]` с каскадным эффектом (outage, earthquake, и т.п.)|
| none     | финальная реплика, мгновенные подтверждения            |

правило: один блок `[/]`→`[*]` — одна анимация. не комбинируем
spinner+typeout на одной строке.

---

## 4. финальная реплика

см. cheatsheet §7. для deploy дополнительно:

- всегда упоминает итоговое время деплоя (реалистичное: 2s–23m) либо
  явно говорит о его отсутствии («rolled back», «aborted»)
- вторая строка — опциональная мораль / комментарий
- не содержит маркеров, не в кавычках, без emoji

---

## 5. флаги команды

- `deploy` — random по весам (см. §7)
- `deploy --force` — всегда `happy_path`
- `deploy --friday` — всегда `not_friday`
- `deploy --quiet` — анимации выключены, паузы сокращены до 50ms
- `deploy --list` — выводит список всех сценариев и весов, без проигрывания

---

## 6. правила выбора маркеров (quick)

| в deploy-контексте               | знак  |
|----------------------------------|-------|
| тесты / сборка / пуш прошли      | `[*]` |
| шаг идёт прямо сейчас            | `[/]` |
| осознанно пропустили (twitter, linter, analytics) | `[=]` |
| rate-limit, ждём retry           | `[-]` |
| прод-деплой завершён успешно     | `[*]` |
| прод-деплой abort / rollback     | `[x]` |
| bundle больше обычного, но ок    | `[!]` |
| incident rate вырос              | `[!]` |
| session died (power, network)    | `[x]` |
| человек реплика в диалоге        | `[>]` |
| листенер / идёт мониторинг       | `[:]` |
| PR на ревью, решение не принято  | `[?]` |

итого: 100%. `perfect_day` — rarest.

### cooldown

`perfect_day` не триггерится, если предыдущий вызов был один из:
`rollback`, `hi_mom`, `power_outage`, `dns_the_usual_suspect`. плохие
дни подряд правдоподобнее.
