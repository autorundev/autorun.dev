# autorun :: state & motion cheatsheet

Краткая шпаргалка для сценариев терминала (deploy, help, error states и т.д.).
Полные спеки: `autorun_brand_system_spec.md`, `autorun_motion_spec.md`.

---

## 1. алфавит (13 знаков)

### brand layer (внешний, 5)

| знак  | имя       | смысл                              | цвет            |
|-------|-----------|------------------------------------|-----------------|
| `[*]` | core      | done / ok / identity / active      | `--state-core` cyan   |
| `[>]` | action    | **только user input / prompt**     | `--state-action` fg-0 |
| `[:]` | presence  | attached, listening, ready         | `--state-presence` violet |
| `[!]` | alert     | warning, drift, anomaly            | `--state-alert` amber |
| `[x]` | stop      | fail, abort, terminated            | `--state-stop` red    |

### product layer (внутренний, 8)

| знак  | имя         | смысл                                | цвет                  |
|-------|-------------|--------------------------------------|-----------------------|
| `[_]` | idle        | standby, не начинали                 | `--fg-2` ghost        |
| `[.]` | ping        | heartbeat, низкая активность         | `--fg-2` ghost        |
| `[-]` | hold        | пауза с намерением вернуться         | `--fg-1` secondary    |
| `[/]` | transition  | spinner frame a (runtime step)       | `--state-core` cyan   |
| `[\]` | transition  | spinner frame b (runtime step)       | `--state-core` cyan   |
| `[+]` | extend      | context attached, capability added   | `--state-live` green  |
| `[?]` | unresolved  | ambiguity, нужно уточнение           | `--state-presence` violet |
| `[=]` | bypass      | skipped / deferred / n/a             | `--fg-2` ghost        |

## 2. жёсткие правила

1. **`[>]` — только user input.** System output начинается с `[*]` или
   другого state-знака. Никогда не `[>] running tests` (это ошибка).
2. **`[*]` — успех / завершение шага.** Не `[x]`. `[x]` — это abort.
3. **Каждый процесс:** `[/]` spinner → `[*]` done (или `[!]` warn / `[x]` fatal).
4. **`[!]` требует внимания.** Если «всё ок, просто не делали» — это `[=]`.
5. **`[x]` ломает контракт.** Если skip безопасный — `[=]`. Тест:
   «ломает ли пропуск шага правильность результата?» да → `[x]`, нет → `[=]`.
6. **Дубль маркеров запрещён.** Две строки подряд с одним и тем же
   знаком = ошибка (кроме `[/]`/`[\]` в spinner-цикле).
7. **Регистры не смешиваем.** System (`[x] aborted`) и human (`сломалось,
   смотрим`) живут на отдельных строках.

## 3. skip / abort / warn — как выбрать

| ситуация                                  | знак   |
|-------------------------------------------|--------|
| шаг прошёл успешно                        | `[*]`  |
| шаг осознанно пропущен, всё ок            | `[=]`  |
| шаг на паузе, вернёмся (rate limit и т.п.)| `[-]`  |
| warning, работаем дальше                  | `[!]`  |
| фатальный abort, шаг/сессия мертва        | `[x]`  |
| нужно решение от человека/агента          | `[?]`  |

## 4. анимации (реализовано)

| имя       | поведение                                           | когда                          |
|-----------|-----------------------------------------------------|--------------------------------|
| `spinner` | `[/][-][\][|][-]`, 120ms/frame, 5 frames = 600ms   | любой `[/]` → `[*]`/`[!]`/`[x]`|
| `dots`    | филлер `........` растёт 120ms/точку до колонки 42  | параллельно со spinner         |
| `flash`   | opacity 0 → 150ms → opacity 1                      | только `[!]` и `[x]`           |
| `shake`   | ±2px 4 раза за 300ms, весь блок терминала           | критичные моменты (редко)      |
| `none`    | появляется мгновенно                                | подтверждения, финал           |

### easing
только `linear` (spinner) и `ease-out` (transitions). всё остальное
запрещено.

## 6. структура строки

```
[X] <text>............................ <suffix>
```

- маркер 3 символа
- пробел
- текст lowercase, без точки в конце
- точки-филлер до колонки 42 (в spinner lines)
- суффикс: результат / число / пусто

примеры:
```
[/] running tests
[*] running tests .................. 142/142
[=] twitter post ................... skipped
[-] rate limit ..................... 30s
[!] drift detected
[x] session aborted
```

## 7. финальная реплика

1–2 строки без маркера, с отступом 0, после `dramatic pause` 900ms.
Lowercase, не повторяет содержимое сценария, комментирует его.

```
deployed in 47s. users happy. founders caffeinated.
```

## 8. быстрый voice-check

перед публикацией любой строки:

1. Это говорит человек или система?
   - человек → `[>]` (если в диалоге) или без маркера (если нарратив)
   - система → state-знак по семантике
2. Если система: это успех, процесс, пауза, пропуск, warning, abort?
   - успех → `[*]`
   - процесс → `[/]`
   - пауза с возвратом → `[-]`
   - осознанный skip → `[=]`
   - warning (продолжаем) → `[!]`
   - abort (всё, конец) → `[x]`
3. Есть ли знак, который точно это описывает? Если нет — **не изобретай новый**,
   переформулируй строку.
