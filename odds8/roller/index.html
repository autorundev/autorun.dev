Roller Odds8

Двухсегментный on-chain prediction market на Solana и BSC. Fast mode для дофаминовых ставок, trader mode для осознанного прогнозного трейдинга, общий vault для LP и маркет-мейкеров.

01 · Аудитории

Четыре группы, одна инфраструктура

Продукт работает одновременно для четырёх непересекающихся сегментов. У каждого свой мотив входа, свой способ монетизации, своя метрика успеха. Общая у них только инфраструктура — контракт, оракул, vault, UI.

01 · GAMBLER

Гэмблер

Приходит за дофаминовым циклом — быстрые ставки, мгновенный результат, минимум трения.

Получает fast mode: 30-секундные блоки, session keys без подтверждений, тап-тап-тап. Паритет UX как у бинарных опционов, но on-chain.

02 · TRADER

Трейдер

Приходит за осознанным прогнозом — фиксированный payout, возможность выйти раньше, параллельные позиции.

Получает trader mode: 5–15-минутные блоки, живой P&L, ранний выход через P2P или vault. Этого нет больше нигде на on-chain.

03 · LP

Поставщик ликвидности

Приходит за real yield — реальная доходность из комиссий и P&L против лузеров, без эмиссии токенов.

Кладёт SOL в vault, получает lp-токен, видит breakdown доходности. В любой момент выход.

04 · BOT

Маркет-мейкер

Приходит за предсказуемым edge — детерминированные формулы, permissionless crank, maker rebate.

Арбитражит перекосы блоков, балансирует стороны, собирает ребейты. Снимает направленный риск с vault'а.

Четыре группы не пересекаются по мотиву, но пересекаются по инфраструктуре. Один контракт монетизируется четырьмя независимыми потоками — в этом экономика продукта.

02 · Rolling-блоки

Блоки идут нахлёстом

Новый блок стартует непрерывно, с фиксированным интервалом. Окно сбора ставок — короткая первая часть блока, экспирация — всё остальное. Поскольку окно сбора короче полной длительности, блоки живут параллельно друг с другом: пока один только открылся для ставок, следующий уже в середине экспирации, а третий вот-вот закроется.

03 · Vault

Vault как контрагент

Вместо того чтобы юзеры делили между собой общий пул ставок, они торгуют против vault'а — пула ликвидности, в который LP кладут SOL и получают реальную доходность. Коэффициенты внутри блока формируются по публичной детерминированной формуле от текущего перекоса: чем сильнее перевес одной стороны, тем привлекательнее противоположная. Payout фиксирован в момент клика. Это бинарный опцион, лотерея остаётся в pari-mutuel.

Следствие: поскольку payouts формульные, оператор для расчётов не нужен. Его роль сводится к permissionless crank'у — триггеру закрытия раунда, который может вызвать любой адрес. Трастовая дыра с контролем timing раундов из security аудита закрывается самой архитектурой.

04 · Ранний выход

Гибридный exit: P2P + vault

В любой момент до экспирации юзер может закрыть позицию или долить. Система работает в два шага: сначала ищет P2P-контрагента внутри блока (кого-то, кто хочет долиться на ту же сторону — это чистое перераспределение без нагрузки на vault), затем остаток закрывается через vault по формульной цене. Bid vault'а на несколько процентов хуже P2P — это одновременно спред, комиссия и стимул искать контрагента.

Ранний выход работает осмысленно только на длинных блоках, где цена позиции имеет динамику. Именно поэтому trader mode живёт в 5–15-минутных блоках — там exit превращается в настоящий инструмент управления позицией — короткие блоки дают только закрытие за секунды до settlement.

07 · Стратегическая позиция

Почему блочная модель, а не Polymarket-подход

Естественный вопрос: если Polymarket делает prediction markets непрерывными — вход в любой момент, выход в любой момент, цена фиксируется рынком — почему Roller Odds8 идёт в сторону фиксированных блоков? Причина — в другой экономике продукта.

Polymarket использует CLOB (central limit order book): юзер покупает YES/NO shares события, цена share сходится к оценке вероятности по мере притока информации. Это работает для event markets (выборы, матчи, макро-события), где есть внешний якорь — факт события, который определённо произойдёт. Для прогнозов цены в непрерывной модели каждая комбинация (strike × expiry) становится отдельным рынком со своим стаканом.

Сравнение по четырём измерениям показывает, что это принципиально разные продукты.

01 · Природа сигнала для юзера
Polymarket (CLOB)Цена share — непрерывный сигнал, дёргается от сделки к сделке. Для трейдера информативно, для гэмблера — белый шум: ничего не происходит между сделками, нет чёткого момента «раунд начался / закончился».
Roller (блоки)Каждый блок — кинематографический опыт: старт, напряжение экспирации, разрешение. Дофаминовый цикл, на котором держится гэмблерский сегмент, возможен только при этой структуре.
02 · Что покупает юзер в момент клика
PolymarketShare с плавающим P&L — цена будет меняться под влиянием потока. Payout при экспирации либо 1, либо 0. Это позиция, которую надо мониторить.
RollerЗафиксированный payout в момент клика — сумма, которую получишь при выигрыше, известна сразу и не переосмыслится чужими ордерами. Для retail аудитории это радикально понятнее: не «0.42 за share», а «поставил 10, получу 18».
03 · Концентрация ликвидности
PolymarketЛиквидность фрагментируется по всем (strike × expiry) парам. Работает для event markets (десятки активных событий), проблематично для прогнозов цены — тысячи тонких стаканов, большинство со смертельными спредами.
RollerЛиквидность концентрируется в 2–3 типах блоков (fast / trader), внутри каждого — один vault как единый контрагент. Плотный рынок с первого дня, не требует десятков маркет-мейкеров в каждом тонком стакане.
04 · Экономика vault и LP
PolymarketVault-like механизм в непрерывной модели превращается в перпетуальный AMM с LVR-проблемой (loss-versus-rebalancing, знакомая у всех AMM). Решаемо, но сложнее и менее предсказуемо для LP.
RollerЧётко определённые окна экспозиции vault'а. LP считает ожидаемый P&L на известном горизонте. Модель ближе к классическому букмекерскому балансированию, где edge понятен и воспроизводим.

Что из этого следует

Блочная модель и continuous CLOB — разные продукты для разных аудиторий. Разная экономика, разные требования к ликвидности. Polymarket — это order book trading для тех, кто умеет торговать. Roller Odds8 — игра с фиксированным payout для тех, кто хочет почувствовать трейдинг, не становясь трейдером.

Если odds8 перейдёт в continuous модель, он столкнётся с тремя проблемами одновременно. Первое — конкуренция с Polymarket на его домашнем поле, где у них кратное преимущество по ликвидности и бренду. Второе — потеря гэмблерского сегмента, который Polymarket структурно не может обслуживать: именно этот сегмент монетизируется лучше всего в вертикали. Третье — потеря собственного нарратива: «первый on-chain продукт с нахлёстом блоков, ранним выходом и vault-контрагентом» превращается в «ещё один CLOB для прогнозов, только на Solana».

Блочная модель с нахлёстом — стратегическая ставка на продукт с другой экономикой и другой аудиторией. Это сознательный выбор, который не стоит «апгрейдить» в Polymarket-like continuous — тогда исчезнет сама причина существования продукта как отдельной сущности.

Trader mode с ранним выходом уже встраивает часть удобства continuous markets внутрь блочной структуры: позиция имеет плавающую стоимость, выйти можно в любой момент через P2P или vault. Этого достаточно для трейдерского UX без сдачи собственного позиционирования. Если через год-два продукт захочет расширяться в event markets (политика, спорт, макро) — continuous модель там уместна, потому что у событий есть внешний якорь. Но это будет второй продукт поверх той же инфраструктуры, не замена блоков.

08 · Как запускать

Порядок построения

Граф зависимостей, не roadmap с датами — что от чего зависит, и с чего начинать, чтобы не переделывать позже.

Каждый следующий шаг опирается на предыдущие. Порядок можно ускорять или растягивать по времени, но не менять местами — математика архитектуры сама его подсказывает.

  1. 01
    Credible commitments контракта. Multisig на upgrade authority, verified build, публикация Anchor IDL, внешний аудит. Без этого профессиональный капитал в vault не придёт — и команде самой страшно будет вливать seed-ликвидность. Предусловие для всего остального.
  2. 02
    Session keys и fast mode. Локальный апгрейд контракта под делегированные подписи, UI без подтверждений. Даёт паритет UX с бинарными опционами на существующем функционале ставок. Не требует vault'а — можно выкатить отдельно.
  3. 03
    Vault-контрагент и формульные коэффициенты. Замена pari-mutuel на LP-pool с публичной формулой. Параллельно — permissionless crank для settlement. Это открывает trader mode и снимает trust-дыру с оператора.
  4. 04
    Rolling-блоки с нахлёстом. Переход от последовательных раундов к непрерывному потоку блоков. Работает только вместе с vault'ом — pari-mutuel плохо совместим с перекрывающимися окнами.
  5. 05
    Ранний выход через P2P + vault. Самая сложная механика: P2P-матчинг внутри блока плюс формульная цена через vault как fallback. Требует готового vault'а и rolling-блоков.
  6. 06
    Maker rebate и дифференцированные комиссии. Финальный слой — превращение комиссионной структуры в инструмент управления поведением. Калибровка делается уже на живых данных после запуска vault'а.
